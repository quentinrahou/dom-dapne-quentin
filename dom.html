voici le document html sur lequel nous allons déposer notre travail

Le document object model est une interface de programmation qui permet de structurer son document à l’image d’un arbre. Il définit la façons dont la structures peut être manipulés par les programmes ( sur le web très utile pour le CSS et le java script)

Il a été mis en place par le W3c afin de normaliser le langage de programmation HTML et ainsi faciliter la vie au développeur qui n’était plus obligé d’adapter leur document à chaque Navigateur. Avant la mise en place de cette interface chaque navigateur avait son propre DOM et traduisait « à sa sauce » les document HTML ce qui rendait le travail de dev relativement difficile car il fallait adapter le code en fonction de la conception de chaque navigateur web.

Le Dom est aujourd’hui devenu très important et la totalité des langages de programmation peuvent être lus par le DOM Aujourd’hui nous sommes à la 4e version du DOM  à approfondir
